# 🔥 Повна інтеграція з Google Firebase

Цей документ описує повний процес: створення бази даних, підключення до додатку, налаштування та опис функціоналу.

---

## 1. Створення проекту Firebase

1. Відкрийте [Firebase Console](https://console.firebase.google.com/)
2. Увійдіть через Google-акаунт
3. Натисніть **«Create a project»** (або «Додати проект»)
4. Введіть назву проекту (наприклад: `class-diary`)
5. Google Analytics — за бажанням (можна вимкнути)
6. Натисніть **«Create project»** → дочекайтеся створення → **«Continue»**

---

## 2. Налаштування Realtime Database

1. У лівому меню оберіть **Build → Realtime Database**
2. Натисніть **«Create Database»**
3. Оберіть розташування: **`europe-west1`** (або найближче до вас)
4. Оберіть **«Start in test mode»** → **«Enable»**

### Правила безпеки

Перейдіть у вкладку **Rules** і вставте:

```json
{
  "rules": {
    "schedule": {
      ".read": true,
      ".write": true
    },
    "homework": {
      ".read": true,
      ".write": true
    },
    "settings": {
      ".read": true,
      ".write": true
    },
    "announcements": {
      ".read": true,
      ".write": true
    }
  }
}
```

Натисніть **«Publish»**.

> ⚠️ Ці правила дозволяють читання та запис усім. Для продуктивного середовища рекомендується обмежити `.write` через Firebase Authentication.

---

## 3. Отримання конфігурації Firebase

1. Перейдіть в **Project settings** (⚙️ біля «Project Overview»)
2. Прокрутіть вниз до секції **«Your apps»**
3. Натисніть іконку `</>` (Web)
4. Введіть назву додатку (наприклад: `class-diary-web`)
5. Скопіюйте об'єкт `firebaseConfig`:

```javascript
const firebaseConfig = {
    apiKey: "ваш-api-key",
    authDomain: "ваш-проект.firebaseapp.com",
    databaseURL: "https://ваш-проект-default-rtdb.europe-west1.firebasedatabase.app",
    projectId: "ваш-проект",
    storageBucket: "ваш-проект.firebasestorage.app",
    messagingSenderId: "123456789",
    appId: "1:123456789:web:abcdef"
};
```

---

## 4. Підключення до додатку

### Спосіб 1: Редагування файлу (рекомендовано)

Відкрийте файл `js/firebase-config.js` та замініть значення на свої:

```javascript
const firebaseConfig = {
    apiKey: "ВАШ_API_KEY",
    authDomain: "ВАШ_AUTH_DOMAIN",
    databaseURL: "ВАШ_DATABASE_URL",
    projectId: "ВАШ_PROJECT_ID",
    storageBucket: "ВАШ_STORAGE_BUCKET",
    messagingSenderId: "ВАШ_SENDER_ID",
    appId: "ВАШ_APP_ID"
};
```

### Спосіб 2: Через адмін-панель

1. Відкрийте `admin.html`
2. Увійдіть з паролем адміністратора
3. У секції **«Налаштування Firebase»** вставте JSON конфігурації
4. Натисніть **«Зберегти»**

> Примітка: цей спосіб зберігає конфігурацію в базу для довідки. Для фактичного підключення необхідно редагувати файл `js/firebase-config.js`.

---

## 5. Налаштування пароля адміністратора

Відкрийте файл `js/admin-credentials.js` та змініть пароль:

```javascript
const ADMIN_PASSWORD = 'ваш_новий_пароль';
```

> Цей пароль потрібен для входу в адмін-панель (`admin.html`).

---

## 6. Опис роботи додатку

### Три режими роботи

| Режим | Доступ | Можливості |
|-------|--------|------------|
| **Переглядач** | Без пароля | Перегляд розкладу та домашніх завдань, навігація тижнями |
| **Редактор** | Пароль редактора | Все з переглядача + редагування уроків, домашніх завдань, D&D |
| **Адміністратор** | Пароль адміна | Налаштування класу, школи, пароля редактора, базового розкладу |

### Функціонал переглядача

- Перегляд розкладу уроків на поточний тиждень
- Перегляд домашніх завдань
- Переключення між тижнями (← →)
- Підсвічування поточного дня

### Функціонал редактора

- Додавання, редагування та видалення уроків
- Заповнення домашніх завдань
- Drag & Drop — перетягування уроків для зміни порядку в межах дня
- Автоматичне збереження змін у Firebase

### Функціонал адміністратора (admin.html)

- Встановлення назви класу та школи
- Управління паролем для режиму редактора
- Редагування базового розкладу (автозаповнення)
- Збереження конфігурації Firebase для довідки
- Скидання поточного розкладу до базового

## 7. Структура даних у Firebase

```
├── schedule/         # Поточний розклад
│   ├── monday: [...]
│   ├── tuesday: [...]
│   ├── wednesday: [...]
│   ├── thursday: [...]
│   └── friday: [...]
│
├── homework/         # Домашні завдання
│   ├── 2025-01-20: [{subject, task}, ...]
│   └── ...
│
├── announcements/    # Оголошення на тиждень
│   ├── 2025-01-20: "текст оголошення"
│   └── ...
│
└── settings/         # Налаштування
    ├── className: "..."
    ├── schoolName: "..."
    ├── editorPassword: "..."
    ├── firebaseConfigJson: "..."
    └── defaultSchedule/
        ├── monday: [...]
        └── ...
```

## 8. Додавання значка на головний екран телефону

### Android (Chrome)

1. Відкрийте сайт щоденника в Chrome
2. Натисніть **⋮** (три точки) у верхньому правому куті
3. Виберіть **«Додати на головний екран»** або **«Add to Home screen»**
4. Введіть назву → **«Додати»**
5. На головному екрані з'явиться значок 📚

### iOS (Safari)

1. Відкрийте сайт щоденника у Safari
2. Натисніть іконку **«Поділитися»** (□↑) внизу екрану
3. Прокрутіть вниз та виберіть **«На Початковий екран»** або **«Add to Home Screen»**
4. Введіть назву → **«Додати»**

### Результат

Після додавання значок відкриватиме сайт як окремий додаток — без адресного рядка браузера, на повний екран.

## ⚠️ Поширені проблеми

| Проблема | Рішення |
|----------|---------|
| «Помилка підключення до бази даних» | Перевірте `databaseURL` у `js/firebase-config.js` |
| Дані не зберігаються | Перевірте правила безпеки в Firebase Console |
| Test mode закінчився | Оновіть правила безпеки у Realtime Database → Rules |
| Пароль не працює | Перевірте `js/admin-credentials.js` для адміна, адмін-панель для редактора |
